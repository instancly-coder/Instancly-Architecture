---
name: UI/UX Patterns
description: Measurable UX rules, UI style catalog, and a pre-delivery checklist for any frontend project. Use alongside frontend-design when building websites, landing pages, dashboards, apps, or any interactive UI.
---

When this skill is active, hold every screen to these rules:

- Touch targets ≥ 44×44px; minimum body copy 14px (16px preferred); line-height ≥ 1.5 for prose.
- Color contrast meets WCAG AA (4.5:1 for body text, 3:1 for large text and UI controls).
- Every form field has a visible label, error state, helper text, and a clear submit affordance.
- Empty states explain what the user is looking at and offer a primary next action.
- Loading states use skeletons or progress, not blank space. Errors offer a retry.
- Keyboard navigation works end-to-end; visible focus rings; logical tab order; trap focus inside modals.
- Respect `prefers-reduced-motion`; never auto-play sound; never hijack scroll.
